## Gap Detail

### Gap 1 — Phase 4: Missing VERIFICATION.md (blocker)

**Severity:** Blocker (unverified phase per workflow rules)

**What:** `04-scheduler` has no VERIFICATION.md. The phase implemented `src/scheduler/runCycle.js` and wired `index.js` with node-cron, but was never formally verified.

**What the integration checker found:** SCHD-01 is fully implemented:
- `index.js`: `SCAN_INTERVAL_MINUTES` env var → cron expression `*/${INTERVAL} * * * *` → `cron.schedule(..., { noOverlap: true })` → `task.execute()` on startup
- `runCycle.js`: iterates all scrapers, reads live pause state from `user_config`, runs scrapers, logs to `scan_log`, calls `detectDeals` + `alertSender.enqueue`
- Fallback to `*/15 * * * *` on invalid interval with error log

**Gap is documentation-only.** The scheduler was built correctly and all downstream phases depend on it functioning — Phases 5–8 all run through `runCycle`. The implementation is verifiable by reading `index.js`.

**Resolution options:**
1. Create `04-VERIFICATION.md` for Phase 4 and update REQUIREMENTS.md checkbox for SCHD-01
2. Accept as tech debt and proceed to complete-milestone — note SCHD-01 as "verified by integration checker, not by phase verification"

### Gap 2 — ALRT-01: Embed title is plain text (tech_debt)

**Severity:** Tech debt (non-blocking — link is functionally present)

**What:** `src/discord/alertSender.js` `buildEmbed()` calls `.setTitle(listing.title.slice(0, 256))` but does not call `.setURL(listing.url)`. The listing URL is delivered only as a Discord Button component via `buildRow(url)`.

**Impact:** The embed title is not clickable. Users must click the "Visa annons" button below the embed to reach the listing. This is functional but departs from the Discord convention where deal-alert bots hyperlink the embed title.

**Fix:** Add `.setURL(alert.listing.url)` to the `EmbedBuilder` chain in `buildEmbed()`.
