    Create `.gitignore` FIRST (before any npm init or git add commands) with the following content:

    ```
    # Database

    # Dependencies
    node_modules/

    # Logs
    *.log
    ```

    Then run `npm init -y` to create package.json. Edit package.json to:
    - Set `"description": "Hardware flip scanner — Swedish marketplace bot"`
    - Set `"main": "index.js"`
    - Do NOT add `"type": "module"` — this project uses CommonJS (require/module.exports)
    - Add `"engines": { "node": ">=24.0.0" }`
    - Add scripts:
      ```json
      "scripts": {
        "start": "node index.js",
        "dev": "cross-env NODE_ENV=development node index.js",
        "db:init": "node src/db/init.js"
      }
      ```

    Install production dependencies:
    ```
    npm install better-sqlite3 pino dotenv discord.js@14 cheerio node-cron zod
    ```

    Install dev dependencies:
    ```
    npm install -D pino-pretty
    ```

    CRITICAL: After `npm install` completes, verify better-sqlite3 loaded successfully:
    ```
    node -e "require('better-sqlite3'); console.log('better-sqlite3 OK')"
    ```

    Do NOT install `windows-build-tools` npm package — it is deprecated.
    Do NOT add `"type": "module"` to package.json — p-queue 9.x is ESM-only and will be loaded via dynamic import() in async scheduler code (Phase 4). The rest of the project uses CommonJS.

<task type="auto" tdd="false">
  <name>Task 2: Create logger module and .env.example</name>
  <files>src/utils/logger.js, .env.example, index.js</files>
  <action>
    Create directory `src/utils/` if it does not exist.

    Create `src/utils/logger.js`:
    ```javascript
    'use strict';
    const pino = require('pino');

    const isDev = process.env.NODE_ENV !== 'production';

    const logger = pino(
      { level: isDev ? 'debug' : 'info' },
      isDev
        ? pino.transport({ target: 'pino-pretty', options: { colorize: true } })
        : process.stdout
    );

    module.exports = logger;
    ```

    Create `.env.example` at the project root:
    ```ini
    # .env.example — commit this file, NEVER commit .env
    DISCORD_TOKEN=your_bot_token_here
    DISCORD_CHANNEL_ID=your_alert_channel_id_here
    GUILD_ID=your_server_id_here
    NODE_ENV=development
    ```

    Create `index.js` at the project root (stub entry point):
    ```javascript
    'use strict';
    require('dotenv').config();   // must be first — loads .env into process.env

    const logger = require('./src/utils/logger');

    logger.info('Scanner starting');
    ```

    Create a `.env` file at the project root for local development (gitignored):
    ```ini
    NODE_ENV=development
    ```

    After creating all files, verify with `git status` that `.env` appears as untracked (not staged). If `.env` appears as "Changes to be committed" or "Changes not staged", run `git rm --cached .env` immediately.
  </action>
