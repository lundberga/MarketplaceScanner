# Milestone v1.0: Hardware Flip Scanner MVP

**Status:** ✅ SHIPPED 2026-03-03
**Phases:** 1–8
**Total Plans:** 19
**Code:** ~2,900 LOC JavaScript

## Overview

Nine phases deliver a single-process Node.js bot that monitors Swedish secondhand marketplaces for below-market hardware, scores each listing against user-defined thresholds and Tradera sold-price comparables, and surfaces deal alerts as Discord embeds. v1.0 covers Phases 1–8 (all bot functionality except Windows process hardening, which ships in v1.1).

## Phases

- [x] **Phase 1: Foundation** — SQLite schema, project structure, logging, dotenv, .gitignore (completed 2026-03-02)
- [x] **Phase 2: Tradera Scraper** — Tradera active listing scraper + sold-comps cache + deduplication pipeline (completed 2026-03-02)

## Milestone Summary

**Key Decisions:**
- Node.js CommonJS (no type:module) — discord.js 14 and better-sqlite3 work cleanly with require(); avoids ESM interop issues
- WAL journal mode set as first pragma before any DDL — ordering is required; cannot toggle after first write
- Absolute DB_PATH via path.resolve(__dirname, ...) — prevents SQLITE_CANTOPEN under pm2 or Task Scheduler CWD changes
- INSERT OR IGNORE for deduplication — preserves first_seen timestamp on re-insert; atomically wraps novel inserts
- noOverlap:true (node-cron v4 built-in) for cycle sequencing — p-queue is ESM-only, incompatible with CJS
- task.execute() fires first cycle immediately on startup — v4 API; runOnInit was removed in node-cron v4
- Blocket uses SSR HTML with a.sf-search-ad-link selector — no __NEXT_DATA__ parsing required
- VintedScraper uses internal JSON API (v2/catalog/items) with session-cookie bootstrap — public, no auth required
- SweclockersScraper uses RSS feed (/feeds/marknad) — simpler and more stable than HTML scraping
- lookupSoldPrice injected as optional third arg to detectDeals — enables unit-testable sold-comps without monkey-patching
- enqueue is fire-and-forget in alertSender — preserves noOverlap cron semantics
- Guild-scoped slash command registration — instant propagation vs 1-hour global delay
- Lazy require() inside command switch cases — startup never fails on missing command files
- DELETE for resume (not value='false') — missing row = not paused; avoids stale config rows

**Issues Resolved:**
- Blocket __NEXT_DATA__ blocker — SSR HTML approach with sf-search-ad-link selector used instead
- Phase 4 VERIFICATION.md gap — created retroactively, implementation confirmed correct by code inspection
- SCHD-01 checkbox unchecked despite implementation being complete — fixed in REQUIREMENTS.md

**Issues Deferred:**
- Phase 9 Hardening (SCHD-02) — pm2 Windows process management deferred to v1.1
- Embed title not hyperlinked (no .setURL()) — link available via Button component; fix in v1.1
- Live Discord/scraper human verifications pending — cannot verify without live credentials
